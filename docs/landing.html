DeepJ

A lightweight, pure-Java Transformer library — build, train, and experiment with GPT, Llama, and DeepSeek-style models, zero dependencies.

☕ Pure Java 🔧 Zero Dependencies ⚡ Metal GPU

Getting started

Installation

Add the GitHub Packages repository and dependency to your pom.xml.

<dependency>
    <groupId>io.github.kirstenali</groupId>
    <artifactId>deepj</artifactId>
    <version>0.5.0-alpha</version>
</dependency>
Why DeepJ

Features

Everything you need to build, train, and deploy Transformer models — in pure Java.

📐

Tensor Engine

2-D tensors with 60+ operations. Pluggable backends — swap between CPU and Metal GPU with one line.

🧱

Layer Primitives

Linear, FNN, LayerNorm, RMSNorm, SwiGLU, Multi-Head Self-Attention — all with full forward & backward.

🦙

Modern Architectures

RoPE, RMSNorm, SwiGLU, MLA — ready-made LlamaTransformerBlock and DeepSeekTransformerBlock with the same primitives used by Llama, Mistral, Qwen, and DeepSeek.

🤖

Three Model Families

GPT (LayerNorm + MHSA), Llama (RMSNorm + RoPE + SwiGLU), and DeepSeek (MLA for 8× smaller KV cache). All extend DecoderOnlyModel, implement the CausalLM interface, and work with one CausalLMTraining.trainer call.

⚡

Metal GPU Backend

Lazy, GPU-resident execution on Apple Silicon. Ops record into a command stream and flush in one batch — up to 36× faster than multithreaded CPU.

🏋️

Training Loop

Pluggable Trainer with EMA loss, early stopping, checkpointing hooks. Factories for supervised and causal LM training.

🔤

Tokenizers

ByteTokenizer for quick experiments, full BPE pipeline for subword tokenization. Both implement the Tokenizer interface.

🧮

AdamW Optimizer

Per-parameter state, bias correction, weight decay. Works with any model exposing parameters().

💾

Binary Persistence

Save & load model weights as compact binary files. No external format dependencies.

Apple Silicon

Metal GPU Backend

Lazy, GPU-resident execution. Tensors stay on the GPU between operations — data only transfers when you need it.

Enable in one line

MetalBackend metal = new MetalBackend();
Tensor.setBackend(metal);

Operations route through the Metal backend entrypoint; GPU kernels run on Metal, while unsupported ops fall back to CPU with on-demand materialization. Tensor state remains coherent across lazy execution and periodic resource release. Reading values on CPU (materialize(), get, set, print, or raw data[][] access) triggers a sync/readback boundary.

How it works

CPU GPU │ │ │ a = random(512, 512) ──upload──▶ GPU buffer A │ b = random(512, 512) ──upload──▶ GPU buffer B │ │ │ c = a.matmul(b) │ record op (no execution yet) │ d = c.relu() │ record op (no execution yet) │ e = d.softmaxRows() │ record op (no execution yet) │ │ │ e.materialize() │ flush all 3 ops in one batch │ ◀──download── GPU buffer E │ read e.data[][] │

No round-trips

For GPU-supported chains, intermediate results stay on GPU

Batched execution

All recorded ops flush in a single GPU command buffer

Staleness tracking

Tensors track CPU/GPU freshness before materialization/release

Examples

Classic MLP (FNN)

FNN mlp = new FNN(
        3,                       // inputSize
        new int[]{16, 16},       // hiddenSizes
        3,                       // outputSize
        GELU::new,               // hiddenActivation
        null,                    // outputActivation
        rnd
);

Trainer trainer = SupervisedTraining.trainer(
        mlp,
        new MSELoss(),
        AdamW.defaultAdamW(3e-3),
        x, y, 123L
);

trainer.train(
        3000,  // maxSteps         – stop after 3 000 gradient updates
        3,     // batchSize        – rows sampled per step
        200,   // logEvery         – print loss every 200 steps
        0.98,  // emaBeta          – smoothing factor for moving-average loss
        1e-6   // targetEmaLoss    – early-stop when smoothed loss falls below this
);

GPT Training + Generation

Tensor.setBackend(new MetalBackend());

GPTConfig cfg = new GPTConfig(
        tok.vocabSize(),
        256,  // maxSeqLen
        512,  // dModel
        4,    // nHeads
        5,    // nLayers
        1024  // dFF
);

GPTModel model = new GPTModel(cfg, 42);
Trainer trainer = CausalLMTraining.trainer(model, ds, 1e-4);

trainer.train(10_000_000, 2, 1, 0.98, 0.01, 25, checkpointHook);

// ── Inference ──
String out = TextGenerator.generate(
        model, tok, cfg,
        "Bob Marley was ",
        200,     // maxNewTokens
        0.1,     // temperature – low = deterministic
        20,      // topK – sample from top 20 tokens
        1234L    // seed
);

Llama Training + Generation

int dModel = 512;
LlamaConfig cfg = new LlamaConfig(
        tok.vocabSize(),
        256,                                   // maxSeqLen
        dModel, 4, 5,                          // dModel, nHeads, nLayers
        LlamaConfig.defaultDFF(dModel)         // ≈ 1408 (8/3 × dModel, rounded)
);

LlamaModel model = new LlamaModel(cfg, 42);
Trainer trainer = CausalLMTraining.trainer(model, ds, 1e-4);
trainer.train(10_000_000, 2, 1, 0.98, 0.01, 25, checkpointHook);

String out = TextGenerator.generate(model, tok, cfg, "Bob Marley was ", 200, 0.1, 20, 1234L);

DeepSeek Training + Generation

int dModel = 512;
DeepSeekConfig cfg = new DeepSeekConfig(
        tok.vocabSize(),
        256,          // maxSeqLen
        dModel, 4, 5, // dModel, nHeads, nLayers
        1024,         // dFF
        dModel / 2,   // qRank  — Q latent dimension
        dModel / 4    // kvRank — KV latent dim (8× smaller KV cache)
);

DeepSeekModel model = new DeepSeekModel(cfg, 42);
Trainer trainer = CausalLMTraining.trainer(model, ds, 1e-4);
trainer.train(10_000_000, 2, 1, 0.98, 0.01, 25, checkpointHook);

String out = TextGenerator.generate(model, tok, cfg, "Bob Marley was ", 200, 0.1, 20, 1234L);
